----------------------------Que es programar-----------------------------------------------------
Programar: Darle instrucciones a algun dispositivo
    Se trata de dividir una instruccion grande en varias chicas
Algoritmo: Serie de instrucciones que le damos a una computadora para realizar una tarea 
Funcion: Es un algoritmo que podemos ejecutar varias veces
Codigo: Serie de instrucciones escritas en un lenguaje que la computadora entiende 
Programa: Se puede componer de distintas funciones 
Indentacion: Es un espacio que se le da a cada linea para definir su jerarquia de posicion
Pseudocodigo: Es un programa creado en lenguaje humano para guiarnos 
----------------------------Variables-----------------------------------------------------
Variables: Son espacios en la memoria de la computadora donde puedo almacenar datos bajo un nombre en comun 
Tipos de datos:
    string: texto 
    booleano: True o False (es un binario, si o no, 0 o 1, true o false)
    int: numero entero (positivo o negativo)
    Uint: numero positivo sin decimales
    float: numero con decimales 
    char: caracter suelto

----------------------------Equipos industriales-----------------------------------------------------
PLC (Controlador Logico Programable): Es un dispositivo que se puede programar con el fin de automatizar tareas especificadas 
    Esta fabricado para durar muchos años
    Esta fabricado para entornos industriales
    Existen cientos de fabriantes de plc 
    Solo se puede programar en uno de 5 lenguajes de programacion preestablecidos segun una norma
PLC vs Arduino:
    PLC 
        estan fabricados para entornos industriales 
        son mas robustos
        soportan altas temperaturas
        tiene durabilidad extensa
    Arduino 
        Son faciles de programar  
        son faciles de conseguir
        son mas baratos
        son sensibles / fragiles
HMI (Interfaz Humano Maquina): es un oeprador que se utiliza para mostrar informacion de un proceso
    Permite controlar y parametrizar cualqueir maquina 
    Por lo general hay un HMI para cada maquina que se usa 
    Se comunica facilemtne con los PLC 
Sensores: Son cualqueir cosa que conectamos a los PLC y otorgan informacion sobre el entorno fisico
    Temperatura, Electricidad, etc 
    Se utiliza para monitoriar el sistema 
    Hay muchos tipos de sensores
Actuadores: Son sistemas que estan conectados y permiten ejercer acciones en base a condiciones fisicas
    Por ejemplo un motor que detiene el proceso si la maquina se calienta
    Trabaja en conjunto con los sensores 
    El PLC es el encargado de dar las ordenes y manejar todo 
Sistemas de Comunicaciones: Switches, Gateways, etc
    son aparatos que permiten transmitir grandes datos de un punto a otro 
SCADA/DCS: Son sistema de visualizacion 
    Son superiores a los HDMI 
    Sirven para monitorear una planta entera o sistemas completos
    SCADA (Sistema de Control y Distribucion): Son para adquisicion y distribucion 
        Monitorea y coordina diferentes sistemas y procesos, asegurando que todo funcione en conjunto
    DCS (Sistema de Control Distribuido): Son para controles
        Controla y optimiza el proceso para asegurar eficiencia, seguridad y calidad
    Se comunican con gran cantidad de sistemas industriales 
    Se utiliza mucho en plantas quimicas o plantas muy grandes con informacion demaciado grande 
Protocolo de comunicacion: Lenguaje con el que las maquinas se van a comunicar entre si 
    Permite que todas las maquinas se puedna comunicar entre si sin importar la marca ni modelo ni nada

----------------------------Protocolos de comunicacion-----------------------------------------------------
Protocolo: Son un conjunto de reglas que utilizan 2 o mas dispositivos para intercambiar informacion entre si 
    "idioma"
    orden
    medio
    etc
MQTT: es para establecer una comunicacion con dispositivos con ancho de badna limitado o conexiones de red limitado 
    Es liviano
    Se usa mucho en domotica e IOT
    Es confiable, se envia una y otra vez la info. hasta que llegue 
    Podemos añadir o quitar equipos sin interrumpir el flujo
    Es faicl de operar 
    Es ligero y podemos enviar mucha data sin problemas de saturacion
Mobdus: es para conectar todo tipo de equipos de la industria
    Es para industria 
    Casi todos los equipos usan mobdus 
    Permite enviar comandos, ver analitifcas, etc 
    Se utiliza para los PLCs
    Tiene documentacion solida
    Practicamente todos los dispositivos usas mobdus 
    Es facil de establecer la comunicacion 
    Es facil de diagnosticar 
RJ45: Es el puerto / conector donde se conecta el cable ethernet
LAN: Red de Area Local (una red de dispositivos sin acceso a internet)
    Pueden haber mas de una LAN en una industria
MAC: Es un tipo de identificador utilizado por el router para saber a que dispositivo entregarle la informacion  
IP: Es un tipo de identificador que permite la comunicacion entre los distintos nodos de la red 
    Dentro de una industria la IP se establece manualmente y es estatica 
    Son 4 numeros separados por puntos (IPV4)
    Por lo general es de /24 --> 255.255.255.0
----------------------------Mobdus-----------------------------------------------------
Mobdus:
    consta de un cliente y un servidor 
        Servidor: Almacena la informacion y la devuelve si la peticion es valida
            Ejemplos:
                PLC 
                Variador de resistencia
                etc
            Cada uno tiene su info. organizada en tablas
                Segun el numero de fila se puede segmentar en 4 partes imporrtantes:
                    Coils: info referida a las salidas digitales de un PLC (de encendido o apagado / 0 o 1 / True o False)
                    Inputs: info que llega al PLC de forma digital
                        sensores 
                        botones 
                        etc 
                    Input Registers: info de los sensores analogicos (que puedan medir mas de dos valores)
                    Holding Resisters: contiene datos que pueden ser modificados por el cliente, como parametros
        Cliente: Se conecta al servidor de uno a la vez y solicita informacion 
            Ejemplo:
                SCADA 
                PLC
                HDMI 
                etc
    Instalar: pip install pymodbus
----------------------------MQTT-----------------------------------------------------
MQTT:
    Es muy util en sistemas de monitoreo
    Consta minimo de 2 entidades:
        Broker: Quien se encarga de gestionar todas las conexiones y se encarga de distribuir los datos a todas las conexiones 
            Es responsable de almacenar y reenviar los mensajes a los clientes suscritos a esos temas
        Cliente/s: Los clientes pueden ser dispositivos IoT, aplicaciones, servidores, etc
            pueden publicar mesajes o suscribirse a ciertos temas:
                Publicar: 
                    enviar la informacion al broker
                    le envia un valor y el topico al que esta asociado
                    es como un log, anota los datos
                Suscribirse: 
                    Un cliente suscriptor se registra en el broker para recibir mensajes sobre un tema específico
                    El cliente define a qué temas desea suscribirse
                    
    Topico: es la categoria o tema del sensor (categoria tempreatura, categoria presion, etc)
    Instalar: pip install paho-mqtt
